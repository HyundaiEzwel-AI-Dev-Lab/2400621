# 내 업무 화면 설계서

## 1. 문서 목적

본 문서는 `docs/01_requirements.md`의 내 업무 요구사항을 구현 가능한 화면 단위로 구체화한다. 설계 기준은 `docs/00_project_context.md`를 따른다.

- Frontend: Vue3, Vue Router, Pinia, Vite, TypeScript
- Nuxt 사용 금지
- Backend: Spring Boot 3 + Java 21
- API: REST API
- 실행 환경: Docker, Docker Compose

| 동작 | `내업무` 선택 시 내 업무 화면으로 이동 |

### 3.2 상단 헤더

| 항목 | 내용 |
| --- | --- |
| 화면명 | `내업무` |
| 브레드크럼 | `> 내업무` |
| 조회 기준 | `통합` |
| 보기 전환 | `캘린더형`, `카드형` |
| 이동 버튼 | 좌우 이동 버튼 |

### 3.3 요약 지표

| 순서 | 지표 | 표시 형식 | 설명 |
| --- | --- | --- | --- |
| 1 | 진행 프로젝트 | `N 건` | 로그인 사용자의 진행 프로젝트 수 |
| 2 | 내 할 일 | `N 건` | 로그인 사용자의 업무 수 |
| 3 | 금주 마감 | `N 건` | 이번 주 마감 업무 수 |
| 4 | 지연 | `N 건` | 마감일 초과 업무 수 |
| 5 | 대기 | `N 건` | 접수 또는 대기 상태 프로젝트 수 |

### 3.4 보기 전환 정책

- 기본 진입 화면은 `카드형`으로 한다.
- 사용자가 `캘린더형`을 선택하면 같은 경로에서 `view=calendar` 상태로 전환한다.
- 사용자가 `카드형`을 선택하면 `view=card` 상태로 전환한다.
- 새로고침 시 URL의 `view` 값을 기준으로 화면을 복원한다.
- `view` 값이 없거나 유효하지 않으면 `card`로 처리한다.

## 4. SCR-MYWORK-001 내 업무 카드형

### 4.1 화면 목적

사용자가 진행 프로젝트, 직접 처리할 업무, 대기 프로젝트를 한 화면에서 빠르게 확인한다.

### 4.2 화면 구성

| 영역 | 구성 요소 | 설명 |
| --- | --- | --- |
| 요약 지표 | 진행 프로젝트, 내 할 일, 금주 마감, 지연, 대기 | 전체 현황 표시 |
| 진행중 | 프로젝트 카드 목록 | 진행 중 프로젝트 표시 |
| 내 할 일 | 업무 목록 테이블 | 사용자 업무 목록 표시 |
| 대기 | 대기 프로젝트 카드 목록 | 접수/대기 프로젝트 표시 |
| 안내 | 안내 문구 | WBS 및 일정 표시 정책 안내 |

### 4.3 진행중 프로젝트 영역

| 항목 | 표시 조건 | 설명 |
| --- | --- | --- |
| 영역 제목 | 항상 | `진행중 (N)` |
| 프로젝트 카드 | 진행 프로젝트가 1건 이상 | 프로젝트별 카드 표시 |
| 빈 상태 | 진행 프로젝트 0건 | `배정된 프로젝트가 없습니다.` |
| 좌우 이동 | 카드가 화면 표시 수보다 많을 때 | 카드 목록 탐색 |

프로젝트 카드 표시 데이터:

| 항목 | 예시 |
| --- | --- |
| 상태 | 테스트, 협의중, 처리중 |
| 기준일/D-day | `0000/00/00 (D-14)` |
| 프로젝트명 | `주문취소 시 쿠폰 할인취소 정보 ...` |
| 공정률 | `80%` |
| 관련 건수 | `6`, `9`, `8` |

### 4.4 내 할 일 영역

| 항목 | 표시 조건 | 설명 |
| --- | --- | --- |
| 영역 제목 | 항상 | `내 할 일 (N)` |
| 테이블 | 내 할 일 1건 이상 | 로그인 사용자에게 배정된 WBS 업무 전체 표시 |
| 빈 상태 | 내 할 일 0건 | `진행중인 업무가 없습니다.` |
| 더보기 버튼 | 각 업무 행 | 업무별 후속 메뉴 표시 |

테이블 컬럼:

| 컬럼 | 표시 형식 | 설명 |
| --- | --- | --- |
| 업무명 | 텍스트 | WBS 업무명 |
| 마감일(D-day) | `M/D 마감 (D-N)` 또는 `일정 미등록` | 일정 등록 여부 및 마감 상태 |
| 공정률 | `N%` 또는 `-%` | 업무 진행률 |
| 프로젝트명 | 텍스트 말줄임 | 소속 프로젝트명 |
| 액션 | `...` | 더보기 메뉴 |

표시 정책:

- 일정 등록 업무는 마감일과 D-day를 표시한다.
- 일정 미등록 업무는 마감일 칸에 `일정 미등록`, 공정률 칸에 `-%`를 표시한다.
- 캘린더 본문에는 일정 등록 업무만 표시한다.
- 카드형 `내 할 일`의 총 건수는 일정 등록/미등록 업무를 모두 포함한다.

### 4.5 대기 영역

| 항목 | 표시 조건 | 설명 |
| --- | --- | --- |
| 영역 제목 | 항상 | `대기 (N)` |
| 대기 카드 | 대기 프로젝트 1건 이상 | 접수/대기 프로젝트 표시 |
| 빈 상태 | 대기 프로젝트 0건 | `접수된 프로젝트가 없습니다.` |
| 좌우 이동 | 카드가 화면 표시 수보다 많을 때 | 대기 프로젝트 탐색 |

대기 카드 표시 데이터:

| 항목 | 예시 |
| --- | --- |
| 상태 | 접수 |
| 프로젝트명 | 전사 프로젝트 관리 시스템 구축 |
| 담당 조직 | 테크담당, 상품사업부 |
| 예정일 | `0000/00/00 (D-00)`, `오픈일 미정` |

### 4.6 안내 영역

표시 문구:

- 프로젝트 관리 > 요구사항 관리에서 요구사항이 `확정`되면 WBS 관리 메뉴에 작업범위가 자동 생성됩니다.
- WBS 관리 메뉴에서 일정이 등록된 업무만 내 할 일 업무에 일정이 표시됩니다.
- 단, 캘린더에는 일정 등록된 업무만 표시됩니다.

## 5. SCR-MYWORK-002 내 업무 캘린더형

### 5.1 화면 목적

사용자가 월 단위로 내 업무 일정을 확인하고, 일정 미등록 업무를 별도로 관리한다.

### 5.2 화면 구성

| 영역 | 구성 요소 | 설명 |
| --- | --- | --- |
| 요약 지표 | 진행 프로젝트, 내 할 일, 금주 마감, 지연, 대기 | 카드형과 동일 |
| 월 헤더 | 이전, 기준 월, 다음, 오늘 | 조회 월 제어 |
| 캘린더 | 월간 날짜 그리드 | 일정 등록 업무 표시 |
| 일정 미등록 업무 | 우측 패널 | 캘린더에 표시되지 않는 업무 표시 |

### 5.3 월 헤더

| 항목 | 동작 |
| --- | --- |
| 이전 버튼 | 이전 월로 이동 |
| 기준 월 | `YYYY년 M월` 형식 표시 |
| 다음 버튼 | 다음 월로 이동 |
| 오늘 버튼 | 현재 날짜가 포함된 월로 이동 |

### 5.4 캘린더 그리드

| 항목 | 설명 |
| --- | --- |
| 표시 단위 | 월간 |
| 날짜 셀 | 일자별 업무 일정 표시 |
| 일정 항목 | 업무명, 기간 또는 마감일, 프로젝트명 표시 |
| 지연 배지 | 지연 일정에 `지연` 표시 |
| 더보기 | 셀 내 표시 한도 초과 시 `+N 더보기` 표시 |

### 5.5 일정 미등록 업무 패널

| 항목 | 표시 조건 | 설명 |
| --- | --- | --- |
| 패널 제목 | 항상 | `일정 미등록 업무 (N)` |
| 업무 목록 | 일정 미등록 업무 1건 이상 | 프로젝트명, WBS 업무명, 일정 등록 버튼 표시 |
| 빈 상태 | 일정 미등록 업무 0건 | `일정 미등록 업무가 없습니다.` |
| 스크롤 | 목록이 패널 높이를 초과할 때 | 패널 내부 스크롤 |

프로토타입 정책:

- `일정 등록` 버튼은 일정관리 화면으로 이동한다.
- 본 프로토타입에서는 일정 저장, 수정, 삭제 화면은 구현 범위에 포함하지 않는다.

## 6. SCR-MYWORK-003 날짜별 내 업무 팝업

### 6.1 표시 조건

- 캘린더 날짜 셀의 `+N 더보기`를 선택한다.
- 특정 날짜의 업무 상세 목록 진입 동작이 발생한다.

### 6.2 화면 구성

| 항목 | 설명 |
| --- | --- |
| 제목 | `M/D 내 업무 (N건)` |
| 닫기 버튼 | 팝업 닫기 |
| 업무 목록 | 업무명, 마감일, 프로젝트명 표시 |
| 스크롤 | 목록이 팝업 높이를 초과할 때 표시 |

### 6.3 동작

- 팝업 바깥 영역 또는 닫기 버튼 선택 시 닫는다.
- ESC 키로 닫을 수 있다.
- 팝업이 열린 동안 배경 화면의 스크롤은 제한한다.

## 7. SCR-MYWORK-005 업무 행 더보기 메뉴

## 8. 상태 정의

| 상태 | 표시 방식 |
| --- | --- |
| 로딩 | 화면 주요 영역에 로딩 표시 |
| 데이터 있음 | 요약 지표 및 각 영역 데이터 표시 |
| 진행 프로젝트 없음 | `배정된 프로젝트가 없습니다.` |
| 내 할 일 없음 | `진행중인 업무가 없습니다.` |
| 대기 프로젝트 없음 | `접수된 프로젝트가 없습니다.` |
| 일정 미등록 업무 없음 | `일정 미등록 업무가 없습니다.` |
| API 오류 | 재시도 가능한 오류 안내 표시 |

## 8.1 데이터 로딩 정책

| 화면 | 기본 API | 보조 API | 설명 |
| --- | --- | --- | --- |
| 카드형 | `GET /api/v1/my-work/card` | 없음 | 요약 지표, 진행 프로젝트, 내 할 일, 대기 프로젝트를 한 번에 조회 |
| 캘린더형 | `GET /api/v1/my-work/calendar` | `GET /api/v1/my-work/calendar/daily` | 요약 지표, 캘린더 일정, 일정 미등록 업무 패널을 한 번에 조회 |
| 날짜별 팝업 | `GET /api/v1/my-work/calendar/daily` | 없음 | 선택 날짜의 전체 업무 목록 조회 |

캘린더형 화면은 초기 진입 시 별도의 일정 미등록 업무 API를 추가 호출하지 않는다. 일정 미등록 업무 목록은 캘린더 API 응답에 포함한다.

## 9. 접근성 및 사용성

- 보기 전환, 월 이동, 오늘, 더보기, 닫기, 일정 등록 버튼은 키보드로 조작 가능해야 한다.
- `지연` 상태는 색상뿐 아니라 텍스트 배지로 표시한다.
- 팝업은 포커스 트랩을 적용한다.
- 긴 프로젝트명과 업무명은 말줄임 처리하되 전체 정보 확인 수단을 제공한다.

## 10. 구현 제외

본 문서는 설계 문서이며 구현은 포함하지 않는다. 다음 항목은 본 단계에서 구현하지 않는다.

- Vue 컴포넌트 생성
- API Controller/Service/Repository 코드 생성
- DB 마이그레이션 파일 생성
- Docker 설정 파일 생성
