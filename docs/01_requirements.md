# 기능 목록

## 1. 내 업무 진입 및 공통 화면

- 좌측 메뉴에서 `내업무` 메뉴를 선택해 내 업무 화면으로 진입한다.
- 상단에는 현재 화면명 `내업무`와 브레드크럼 `> 내업무`를 표시한다.
- `통합` 기준으로 내 업무를 조회한다.
- 화면 우측 상단에서 `카드형`과 `캘린더형` 보기 방식을 전환한다.
- 카드형/캘린더형 화면 모두 공통 요약 지표를 표시한다.
  - 진행 프로젝트
  - 내 할 일
  - 금주 마감
  - 지연
  - 대기
- 요약 지표는 건수 단위로 표시한다. 예: `진행 프로젝트 5건`, `내 할 일 4건`, `금주 마감 2건`, `지연 1건`, `대기 2건`
- 화면 또는 패널 단위로 좌우 이동 버튼을 제공한다.

## 2. 카드형 보기

- 카드형 보기에서는 `진행중`, `내 할 일`, `대기` 영역을 표시한다.
- `진행중` 영역에는 진행 중인 프로젝트 카드를 표시한다.
- 진행 중인 프로젝트가 없으면 `배정된 프로젝트가 없습니다.`를 표시한다.
- 프로젝트 카드는 다음 정보를 표시한다.
  - 프로젝트 상태
  - 기준일 또는 마감일
  - D-day
  - 프로젝트명
  - 공정률
  - 관련 건수 지표
- 프로젝트 상태 예시는 `테스트`, `협의중`, `처리중`이다.
- 프로젝트 카드 목록은 좌우 이동으로 탐색할 수 있다.
- `내 할 일` 영역에는 일정이 등록된 내 업무 목록을 표 형태로 표시한다.
- 내 할 일이 없으면 `진행중인 업무가 없습니다.`를 표시한다.
- 내 할 일 목록은 다음 컬럼을 표시한다.
  - 업무명
  - 마감일(D-day)
  - 공정률
  - 프로젝트명
- 지연 업무는 `지연` 배지를 표시한다.
- 일정이 없는 업무는 마감일에 `일정 미등록`, 공정률에 `-%`로 표시한다.
- 각 업무 행에는 더보기 메뉴를 제공한다.
- 더보기 메뉴에서는 `일정관리`, `WBS 상세`로 이동할 수 있다.
- `대기` 영역에는 접수 또는 대기 상태의 프로젝트 카드를 표시한다.
- 대기 프로젝트가 없으면 `접수된 프로젝트가 없습니다.`를 표시한다.
- 대기 프로젝트 카드는 다음 정보를 표시한다.
  - 프로젝트명
  - 상태
  - 담당 조직 또는 부서
  - 예정일 또는 오픈일
- 예정일이 없는 경우 `오픈일 미정`으로 표시한다.

## 3. 캘린더형 보기

- 캘린더형 보기에서는 월간 캘린더와 일정 미등록 업무 패널을 함께 표시한다.
- 캘린더 상단에는 기준 월을 표시한다. 예: `2026년 3월`
- 이전/다음 월 이동 기능을 제공한다.
- `오늘` 버튼을 제공한다.
- 일정이 등록된 업무는 캘린더 날짜 영역에 표시한다.
- 캘린더 일정 항목은 업무명, 마감일 또는 기간, 프로젝트명을 함께 표시한다.
- 지연된 일정 항목은 `지연` 배지를 표시한다.
- 한 날짜에 표시할 일정이 많으면 `+2 더보기`, `+3 더보기`와 같이 더보기 링크를 표시한다.
- 더보기 또는 날짜 선택 시 해당 날짜의 내 업무 팝업을 표시한다.
- 날짜별 내 업무 팝업은 다음 정보를 표시한다.
  - 선택 날짜
  - 해당 날짜의 내 업무 건수
  - 업무명
  - 마감일
  - 프로젝트명
  - 닫기 버튼
  - 스크롤
- 일정 미등록 업무 패널에는 일정이 없는 업무 목록을 표시한다.
- 일정 미등록 업무가 없으면 `일정 미등록 업무가 없습니다.`를 표시한다.
- 일정 미등록 업무가 있으면 업무별로 프로젝트명, WBS 업무명, `일정 등록` 버튼을 표시한다.
- 일정 미등록 업무 패널은 항목이 많을 경우 스크롤한다.

## 4. 안내 문구

- 카드형 화면 하단에는 다음 안내를 표시한다.
  - 프로젝트 관리 > 요구사항 관리에서 요구사항이 `확정`되면 WBS 관리 메뉴에 작업범위가 자동 생성된다.
  - WBS 관리 메뉴에서 일정이 등록된 업무만 내 할 일 업무에 일정이 표시된다.
  - 단, 캘린더에는 일정 등록된 업무만 표시된다.

# 사용자 시나리오

## 1. 내 업무 현황 확인

1. 사용자가 H-PMS에 접속한다.
2. 좌측 메뉴에서 `내업무`를 선택한다.
3. 사용자는 상단 요약 지표에서 진행 프로젝트, 내 할 일, 금주 마감, 지연, 대기 건수를 확인한다.
4. 사용자는 현재 처리해야 할 업무량과 지연 여부를 판단한다.

## 2. 카드형에서 진행 프로젝트 확인

1. 사용자가 `카드형` 보기를 선택한다.
2. 사용자는 `진행중` 영역에서 진행 중인 프로젝트 카드를 확인한다.
3. 프로젝트 카드에서 상태, D-day, 프로젝트명, 공정률을 확인한다.
4. 진행 프로젝트가 여러 개이면 좌우 이동 버튼으로 다른 프로젝트를 탐색한다.

## 3. 카드형에서 내 할 일 처리

1. 사용자가 카드형 화면의 `내 할 일` 영역을 확인한다.
2. 업무명, 마감일(D-day), 공정률, 프로젝트명을 기준으로 처리 우선순위를 판단한다.
3. 지연 배지가 있는 업무를 우선 확인한다.
4. 특정 업무의 더보기 메뉴를 연다.
5. 사용자는 `일정관리` 또는 `WBS 상세`로 이동해 후속 업무를 처리한다.

## 4. 대기 프로젝트 확인

1. 사용자가 카드형 화면의 `대기` 영역을 확인한다.
2. 접수 상태의 프로젝트명, 담당 조직, 예정일 또는 오픈일을 확인한다.
3. 대기 프로젝트가 여러 개이면 좌우 이동 버튼으로 탐색한다.
4. 대기 프로젝트가 없으면 `접수된 프로젝트가 없습니다.` 안내를 확인한다.

## 5. 캘린더형에서 월간 업무 일정 확인

1. 사용자가 `캘린더형` 보기를 선택한다.
2. 사용자는 기준 월의 업무 일정을 캘린더에서 확인한다.
3. 이전/다음 월 이동 또는 `오늘` 버튼으로 조회 기준을 변경한다.
4. 일정 항목의 업무명, 기간, 프로젝트명을 확인한다.
5. 지연 배지가 있는 일정을 확인한다.

## 6. 특정 날짜의 업무 더보기

1. 사용자가 캘린더에서 `+N 더보기`를 선택한다.
2. 선택한 날짜의 내 업무 팝업이 열린다.
3. 사용자는 해당 날짜의 업무 건수와 업무 목록을 확인한다.
4. 목록이 길면 스크롤로 추가 업무를 확인한다.
5. 확인 후 닫기 버튼으로 팝업을 닫는다.

## 7. 일정 미등록 업무에 일정 등록

1. 사용자가 캘린더형 화면 우측의 `일정 미등록 업무` 패널을 확인한다.
2. 일정 미등록 업무가 있으면 프로젝트명과 WBS 업무명을 확인한다.
3. 사용자는 `일정 등록` 버튼을 선택한다.
4. 시스템은 해당 업무의 일정관리 화면으로 이동한다.
5. 실제 일정 저장 및 저장 후 캘린더 반영은 본 프로토타입 범위에서 제외한다.

## 8. 업무가 없는 상태 확인

1. 사용자가 내 업무 화면에 진입한다.
2. 배정된 진행 프로젝트가 없으면 `배정된 프로젝트가 없습니다.`를 확인한다.
3. 내 할 일이 없으면 `진행중인 업무가 없습니다.`를 확인한다.
4. 접수된 프로젝트가 없으면 `접수된 프로젝트가 없습니다.`를 확인한다.
5. 일정 미등록 업무가 없으면 `일정 미등록 업무가 없습니다.`를 확인한다.

# 화면 정의

## 1. 내 업무 카드형 화면

- 화면 목적: 사용자가 진행 프로젝트, 직접 처리할 내 할 일, 대기 프로젝트를 한 화면에서 확인한다.
- 주요 영역:
  - 좌측 메뉴
  - 화면명 및 브레드크럼
  - 보기 전환 탭: 카드형, 캘린더형
  - 요약 지표 영역
  - 진행중 프로젝트 카드 영역
  - 내 할 일 목록 영역
  - 대기 프로젝트 카드 영역
  - 안내 문구 영역
- 주요 상태:
  - 데이터 있음
  - 진행 프로젝트 없음
  - 내 할 일 없음
  - 대기 프로젝트 없음
- 주요 액션:
  - 카드형/캘린더형 전환
  - 프로젝트 카드 좌우 이동
  - 업무 행 더보기
  - 일정관리 이동
  - WBS 상세 이동

## 2. 내 업무 카드형 상세 목록 화면

- 화면 목적: 카드형 화면에서 내 할 일과 대기 프로젝트를 더 많이 확인한다.
- 주요 영역:
  - 내 할 일 목록
  - 대기 프로젝트 목록
  - 업무 행별 더보기 버튼
  - 더보기 메뉴
  - 안내 문구
- 주요 액션:
  - 업무별 `일정관리` 이동
  - 업무별 `WBS 상세` 이동
  - 대기 프로젝트 좌우 이동

## 3. 내 업무 캘린더형 화면

- 화면 목적: 사용자가 월간 기준으로 내 업무 일정을 확인한다.
- 주요 영역:
  - 좌측 메뉴
  - 화면명 및 브레드크럼
  - 보기 전환 탭: 카드형, 캘린더형
  - 요약 지표 영역
  - 월 이동 헤더
  - 오늘 버튼
  - 월간 캘린더
  - 일정 미등록 업무 패널
- 주요 상태:
  - 캘린더 일정 있음
  - 일정 더보기 있음
  - 지연 일정 있음
  - 일정 미등록 업무 없음
  - 일정 미등록 업무 있음
- 주요 액션:
  - 이전 월 이동
  - 다음 월 이동
  - 오늘 이동
  - 일정 더보기 열기
  - 일정 등록

## 4. 날짜별 내 업무 팝업

- 화면 목적: 캘린더에서 특정 날짜에 몰린 내 업무를 목록으로 확인한다.
- 표시 조건: 캘린더 날짜에서 `+N 더보기` 또는 날짜별 상세 진입을 선택한다.
- 주요 영역:
  - 선택 날짜
  - 내 업무 건수
  - 업무 목록
  - 닫기 버튼
  - 스크롤 영역
- 주요 액션:
  - 팝업 닫기
  - 스크롤로 추가 업무 확인

## 5. 일정 미등록 업무 패널

- 화면 목적: 캘린더에 표시되지 않는 일정 미등록 업무를 별도로 확인하고 일정 등록을 유도한다.
- 주요 영역:
  - 패널 제목 및 건수
  - 업무 목록
  - 업무별 프로젝트명
  - 업무별 WBS 업무명
  - 업무별 일정 등록 버튼
  - 빈 상태 문구
- 주요 액션:
  - 일정 등록
  - 스크롤

# 데이터 정의

## 1. 요약 지표 데이터

| 데이터 | 설명 | 예시 |
| --- | --- | --- |
| 진행 프로젝트 수 | 사용자에게 배정된 진행 중 프로젝트 수 | 5건 |
| 내 할 일 수 | 사용자가 처리해야 할 업무 수 | 4건 |
| 금주 마감 수 | 이번 주 마감 대상 업무 수 | 2건 |
| 지연 수 | 마감일을 초과한 업무 수 | 1건 |
| 대기 수 | 접수 또는 대기 상태 프로젝트 수 | 2건 |

## 2. 프로젝트 카드 데이터

| 데이터 | 설명 | 예시 |
| --- | --- | --- |
| 프로젝트 ID | 프로젝트 식별자 | 미표시 |
| 프로젝트명 | 프로젝트 카드에 표시되는 이름 | DL이앤씨 바우처 정책 변경 |
| 프로젝트 상태 | 현재 프로젝트 진행 상태 | 테스트, 협의중, 처리중, 접수 |
| 기준일 | 프로젝트 기준 날짜 또는 마감일 | 0000/00/00 |
| D-day | 기준일 대비 남은 일수 또는 초과 일수 | D-14, D-24, D-37 |
| 공정률 | 프로젝트 진행률 | 80%, 30%, 55% |
| 담당 조직 | 대기 프로젝트의 담당 조직 또는 부서 | 테크담당, 상품사업부 |
| 오픈일 | 프로젝트 오픈 예정일 | 0000/00/00, 오픈일 미정 |
| 관련 건수 | 카드 하단에 표시되는 업무/지표 건수 | 6, 9, 8 |

## 3. 내 할 일 데이터

| 데이터 | 설명 | 예시 |
| --- | --- | --- |
| 업무 ID | 업무 식별자 | 미표시 |
| 업무명 | 사용자가 수행할 작업명 | 단위 테스트 |
| 마감일 | 업무 마감일 | 3/20 마감 |
| D-day | 마감일 기준 잔여 또는 초과 일수 | D+2, D-5, D-13 |
| 지연 여부 | 마감일 초과 여부 | 지연 |

# 정책 정의

## 1. WBS 생성 정책

- 프로젝트 관리 > 요구사항 관리에서 요구사항이 `확정`되면 WBS 관리 메뉴에 작업범위가 자동 생성된다.

## 2. 내 할 일 표시 정책

- 카드형 `내 할 일`은 로그인 사용자에게 배정된 WBS 업무 전체를 표시한다.
- WBS 관리 메뉴에서 일정이 등록된 업무는 마감일과 D-day를 표시한다.
- 일정이 등록되지 않은 업무는 카드형 `내 할 일` 목록에서 마감일을 `일정 미등록`, 공정률을 `-%`로 표시한다.
- 캘린더에는 일정이 등록된 업무만 표시된다.
- 캘린더에 표시되지 않는 일정 미등록 업무는 우측 `일정 미등록 업무` 패널에 표시한다.

## 2.1 프로토타입 일정 등록 정책

- 프로토타입에서는 `일정 등록` 버튼 선택 시 실제 일정 저장 API를 호출하지 않는다.
- `일정 등록` 버튼은 해당 업무의 일정관리 화면으로 이동하는 진입점 역할만 한다.
- 일정 저장, 일정 수정, 일정 삭제 기능은 후속 범위로 둔다.

## 3. 캘린더 표시 정책

- 캘린더형 화면은 월 단위로 업무 일정을 표시한다.
- 일정이 등록된 업무만 캘린더 날짜 영역에 표시한다.
- 일정 미등록 업무는 캘린더 본문에 표시하지 않고 우측 `일정 미등록 업무` 패널에 표시한다.
- 특정 날짜에 표시할 업무가 많으면 일부만 노출하고 `+N 더보기`로 나머지를 확인한다.
- 더보기 팝업에는 선택 날짜의 전체 업무 건수를 표시한다.

## 4. 지연 표시 정책

- 마감일이 지난 업무는 `지연` 배지를 표시한다.
- D-day가 초과된 경우 `D+N` 형식으로 표시한다.
- 마감일까지 남은 경우 `D-N` 형식으로 표시한다.
- `D-Day`는 기준일과 마감일이 같은 경우 표시한다.
- 기준일은 사용자의 로컬 날짜가 아니라 서버 기준일을 사용한다.

## 4.1 금주 마감 집계 정책

- `금주 마감`은 서버 기준일이 속한 주의 월요일부터 일요일까지 마감일이 포함된 업무 수로 집계한다.
- 완료된 업무는 `금주 마감`과 `지연` 집계에서 제외한다.
- 일정 미등록 업무는 마감일이 없으므로 `금주 마감`과 `지연` 집계에서 제외한다.

## 5. 빈 상태 표시 정책

- 진행 프로젝트가 0건이면 `배정된 프로젝트가 없습니다.`를 표시한다.
- 내 할 일이 0건이면 `진행중인 업무가 없습니다.`를 표시한다.
- 대기 프로젝트가 0건이면 `접수된 프로젝트가 없습니다.`를 표시한다.
- 일정 미등록 업무가 0건이면 `일정 미등록 업무가 없습니다.`를 표시한다.

## 6. 이동 정책

- 업무 행의 더보기 메뉴에서 `일정관리`를 선택하면 해당 업무의 일정 관리 화면으로 이동한다.
- 업무 행의 더보기 메뉴에서 `WBS 상세`를 선택하면 해당 업무의 WBS 상세 화면으로 이동한다.
- 일정 미등록 업무의 `일정 등록` 버튼을 선택하면 해당 업무에 일정을 등록할 수 있는 화면 또는 기능으로 이동한다.

# 가정 사항

- PPTX 22~26페이지는 `내업무` 기능의 화면 설계 범위로 본다.
- 페이지 22~24는 카드형 보기, 페이지 25~26은 캘린더형 보기로 본다.
- `통합`은 내 업무 조회 기준 또는 조직/프로젝트 범위 선택값으로 본다.
- `진행 프로젝트`, `내 할 일`, `금주 마감`, `지연`, `대기`는 로그인 사용자 기준의 요약 지표로 본다.
- 카드 하단의 숫자 지표는 프로젝트별 관련 업무 또는 상태별 건수로 추정하되, 정확한 의미는 미확정으로 둔다.
- `내 할 일`은 WBS 업무 중 사용자에게 배정된 업무를 의미한다고 본다.
- 캘린더 일정은 업무의 시작일/종료일 또는 마감일을 기준으로 표시한다고 본다.
- `일정 등록`은 일정관리 화면으로 이동하는 기능으로 본다.
- D-day와 지연 판정은 서버 기준일을 사용하며, 휴일/영업일 보정은 적용하지 않는다.
- 본 문서는 화면 분석 기반 요구사항이며, 실제 API/DB 구조는 아직 확정되지 않은 것으로 본다.

# 미확정 사항

- H-PMS의 실제 사용자 역할과 권한 체계
- `통합` 선택값의 정확한 의미와 선택 가능한 범위
- 프로젝트 카드 하단 숫자 지표의 정확한 의미
- 프로젝트 상태값 전체 목록
- 업무 상태값 전체 목록
- 공정률 산정 방식
- 실제 운영에서 휴일/영업일 보정이 필요한지 여부
- 일정 미등록 업무가 카드형 내 할 일 목록에 포함되는지 여부
- 캘린더 일정의 기간 표시 기준
- `+N 더보기` 팝업에서 업무 선택 시 상세로 이동하는지 여부
- `일정관리`, `WBS 상세`의 운영 환경 최종 URL 또는 화면 ID
- 데이터 로딩 실패, 권한 없음, 삭제된 업무 등 예외 상태 처리 방식
- 모바일 또는 작은 화면에서 카드형/캘린더형을 어떻게 표시할지 여부
- 실제 구현 시 사용할 프론트엔드/백엔드 기술 스택
